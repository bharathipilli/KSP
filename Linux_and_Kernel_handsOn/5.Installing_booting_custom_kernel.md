# Installing & Booting Custom Kernels

## 1. Definition
Installing and booting a custom kernel means **taking the kernel you compiled** and making your system use it at startup.  
This includes:
- Copying kernel images to `/boot/`
- Installing modules
- Updating the bootloader (GRUB, LILO, etc.)
- Rebooting into the new kernel

---

## 2. Why is it important?
- To test **custom features** or patches you added.
- To optimize kernel for **performance** or **hardware compatibility**.
- To debug or develop new **drivers/modules**.
- To use **latest upstream kernel** (new features, security fixes).

---

## 3. Key Files Installed
When you run `make install`, the following go into `/boot/`:
- **bzImage** → compressed bootable kernel (`/boot/vmlinuz-<version>`)
- **System.map** → symbol lookup table (`/boot/System.map-<version>`)
- **initramfs/initrd** → initial RAM disk used during boot
- **config** → saved kernel configuration (`/boot/config-<version>`)

Modules are installed under:
```

/lib/modules/\<kernel\_version>/

````

---

## 4. Step-by-Step Installation & Boot

### Step 1: Install Kernel Modules
```bash
sudo make modules_install
````

This copies `.ko` (kernel object) files into `/lib/modules/`.

### Step 2: Install the Kernel Image

```bash
sudo make install
```

* Places kernel image in `/boot/`
* Creates symbolic links (like `/boot/vmlinuz`)
* Prepares `initramfs` if required

### Step 3: Update Bootloader

For systems with **GRUB**:

```bash
sudo update-grub
```

This scans `/boot/` for new kernels and updates the GRUB menu.

### Step 4: Reboot & Select Kernel

```bash
reboot
```

* At boot, GRUB menu will show multiple kernels.
* Select your **new custom kernel**.
* Use `uname -r` after login to confirm:

```bash
uname -r
```

---

## 5. Example Workflow

```bash
cd linux-6.5
sudo make modules_install
sudo make install
sudo update-grub
reboot
```

After reboot:

```bash
uname -r   # should display your new kernel version
```

---

## 6. Troubleshooting

* **System doesn’t boot?**

  * Boot into the old kernel from GRUB.
  * Recheck your kernel config.
* **Kernel modules missing?**

  * Run `make modules_install` again.
* **Custom kernel not in GRUB menu?**

  * Run `sudo update-grub` or manually edit `/boot/grub/grub.cfg`.

---

## 7. In Short

1. Install modules → `make modules_install`
2. Install kernel → `make install`
3. Update bootloader → `update-grub`
4. Reboot and select new kernel
5. Verify with `uname -r`

✅ You are now booting into your **custom-built Linux kernel**!
